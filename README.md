# Detecting Bots via Mouse Movement Biometrics

A machine learning system that detects bot activity by analysing mouse movement patterns using a Bidirectional LSTM model. Includes a real-time Flask web demo where users trace a circle and the model predicts Human vs Bot confidence.

## Running the Project

### Step 1 ‚Äî Preprocessing

Run `project_preprocessing.ipynb` top to bottom.

**Output:** `X_train.npy`, `X_val.npy`, `X_test.npy`, `y_train.npy`, `y_val.npy`, `y_test.npy`

Each session is preprocessed into shape `(200, 4)`:

| Feature | Description |
|---|---|
| `timestamp_norm` | Time normalised to [0, 1] |
| `event_type` | move=0, click=1, release=2 |
| `x_norm` | X coordinate √∑ screen width |
| `y_norm` | Y coordinate √∑ screen height |

Split: 70% train ¬∑ 15% val ¬∑ 15% test (stratified)

---

### Step 2 ‚Äî Training

Run `project_training.ipynb` top to bottom.

**Output:** `bot_detector_model.keras`, `training_history.png`

#### Model Architecture

```
Input (200, 4)
    ‚Üì
GaussianNoise(0.01)         ‚Üê augmentation / anti-overfit
    ‚Üì
BiLSTM(64, return_seq=True) ‚Üê captures forward + backward patterns
    ‚Üì
BiLSTM(32, return_seq=False)
    ‚Üì
Dense(32, ReLU) + L2
    ‚Üì
Dropout(0.4)
    ‚Üì
Dense(1, Sigmoid)           ‚Üê outputs P(bot)
```

- **Optimizer:** Adam (lr=3e-4)
- **Loss:** BinaryCrossentropy with label_smoothing=0.1
- **Batch size:** 64
- **Class weights:** computed via `sklearn` to handle 3:1 bot:human imbalance
- **Early stopping:** patience=15, monitoring val_loss

#### Training Results

| Metric | Value |
|---|---|
| Test AUC | 0.871 |
| Test Precision (bot) | 98.8% |
| Test Accuracy | 72.5% |
| Test Recall (bot) | 64.3% |
| Train/Val loss gap | ‚àí0.02 (no overfitting) |

Best weights restored from **epoch 5**. Early stopping triggered at epoch 20.

### Step 4 ‚Äî Live Demo

```bash
python app.py
```

Open `http://localhost:5000` in your browser.

**How it works:**
1. A circle is drawn on the canvas with a moving guide dot
2. User traces the circle with their mouse
3. 200 mouse events are captured (timestamp, event type, x, y)
4. Events are preprocessed identically to training data
5. Model predicts P(bot) ‚Üí displayed as Human % and Bot %

> **Note:** The app works even without a trained model (demo mode with simulated predictions). Place `bot_detector_model.keras` in the same folder as `app.py` to use the real model.

## How the Model Works

The model learns temporal patterns in mouse movement sequences. Humans exhibit natural micro-jitter, acceleration curves, and irregular timing. Bots produce mechanically regular intervals, perfectly straight paths, or unnaturally smooth curves.

The output is a single sigmoid value ‚Äî `P(bot)`. Human probability = `1 - P(bot)`.

```
P(bot) ‚â• 0.5  ‚Üí  BOT
P(bot) < 0.5  ‚Üí  HUMAN
```
